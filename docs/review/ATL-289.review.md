### Warnings

**W1 — StatCard 그리드 레이아웃 불균형** (`admin-ui/app/(admin)/page.tsx:115`)

```tsx
<div className="grid grid-cols-2 lg:grid-cols-4 gap-4">
  {/* 5개 StatCard */}
</div>
```

`lg:grid-cols-4` 레이아웃에 5번째 카드("Today's Cache Hit Rate")가 추가되어 lg 화면에서 두 번째 줄에 혼자 표시된다. `lg:grid-cols-5` 또는 `2xl:grid-cols-5`로 조정하면 더 깔끔한 레이아웃이 가능하다. 기능 결함은 아님.

---

**W2 — `logStore` nil 안전성** (`internal/gateway/handler/admin_cache.go:19`)

```go
func NewAdminCacheHandler(cache *exactcache.Cache, logStore *pgstore.LogStore) *AdminCacheHandler {
    return &AdminCacheHandler{cache: cache, logStore: logStore}
}
```

`logStore`가 nil로 전달되면 `Stats()` 호출 시 패닉 발생. 현재 `main.go`에서 항상 초기화된 값을 전달하므로 실질적 위험은 낮음. nil 가드 또는 인터페이스 추상화로 개선 고려 가능 (필수 아님).

---

### Info

**I1 — `CacheStats()` 단위 테스트 없음** (`internal/store/postgres/log_store.go:210`)
DB 통합 테스트 환경이 없어 쿼리 정확성은 실제 PostgreSQL 환경에서만 검증 가능. 테스트 리포트에서도 명시된 한계.

**I2 — `sub` 중복 표시 가능성** (`page.tsx:128`)
`total == 0`일 때 값은 "N/A"이지만 sub는 "0 / 0 requests"로 표시된다. 기능적으로 문제없으나 UX 관점에서 redundant할 수 있음.

**I3 — API 경로 일치 확인**
`cache.stats()` → `apiFetch("/cache/stats")` → `BASE="/api/admin"` → `/api/admin/cache/stats`
Next.js rewrite → `/admin/cache/stats` ← `r.Get("/admin/cache/stats", ...)` ✅ 경로 일치.
