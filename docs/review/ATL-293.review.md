### Warning

**[W1] `vitest.setup.ts:12` — fetch mock 파일 간 상태 누출 위험**
```ts
global.fetch = vi.fn().mockResolvedValue({ ok: true, json: async () => ({}) })
```
- `global.fetch`가 setup에서 한 번 설정되고, `afterEach`에서 리셋되지 않음
- `Sidebar.test.tsx`는 `beforeEach`에서 `vi.clearAllMocks()`로 처리하지만 `pages.test.tsx`는 미처리
- 테스트가 늘어나 mock 반환값을 변경하면 파일 내 순서 의존성 발생 가능
- **권장**: `vitest.config.ts`에 `clearMocks: true` 또는 `resetMocks: true` 추가

**[W2] `app/(admin)/__tests__/pages.test.tsx` — mock 리셋 누락**
```ts
vi.mock('@/lib/api', () => ({
  usage: { topSpenders: vi.fn().mockResolvedValue([]) },
  ...
}))
// afterEach(() => vi.clearAllMocks()) 없음
```
- 현재는 mock 반환값이 고정이라 문제없음
- 향후 특정 테스트에서 mock 동작을 변경하면 이후 테스트에 영향

### Info

```ts
import { describe, it, expect } from 'vitest'  // globals: true이면 불필요
```

