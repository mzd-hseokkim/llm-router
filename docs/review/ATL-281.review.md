## Fix Verification

### W-1: toggleMutation onError handler — FIXED

File: `admin-ui/app/(admin)/routing/page.tsx`, lines 974–988

Before (initial review): `toggleMutation` had no `onError` handler — toggle failures were silently swallowed.

After (verified):
```typescript
const toggleMutation = useMutation({
  mutationFn: (rule: RoutingRule) => {
    const payload: CreateRoutingRulePayload = {
      name: rule.name,
      priority: rule.priority,
      enabled: !rule.enabled,
      match: rule.match,
      strategy: rule.strategy,
      targets: rule.targets,
    };
    return routingRules.update(rule.id, payload);
  },
  onSuccess: () => qc.invalidateQueries({ queryKey: ["routing-rules"] }),
  onError: (e: Error) => setError(e.message),
});
```

`onError: (e: Error) => setError(e.message)` is present. Toggle errors now surface to the user via the same `error` state used by other mutations. Fix is correct and consistent with the rest of the file.

### W-2: Inline import type replaced with imported name — FIXED

File: `admin-ui/app/(admin)/routing/page.tsx`, line 976

Before (initial review): `const payload: import("@/lib/api").CreateRoutingRulePayload`

After (verified): `const payload: CreateRoutingRulePayload`

`CreateRoutingRulePayload` is imported at line 9 of the file. The redundant inline import type has been removed. Fix is correct.
