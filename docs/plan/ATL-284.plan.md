# ATL-284 Planning Document
## [LLM Router] Admin UI: 프롬프트 관리 페이지 (/prompts)

**작성일**: 2026-02-23
**상태**: In Progress
**Branch**: feature/ATL-284

## 1. Background

LLM Router Admin UI에 프롬프트 관리 페이지(`/prompts`)가 없어 운영 불가 상태다. 백엔드 API(`/admin/prompts/*`)와 `lib/api.ts`의 `prompts` 객체(ATL-280에서 완료)는 이미 완성되어 있으며, UI만 없는 상황이다.

### 현재 상태

| 컴포넌트 | 위치 | 상태 |
|---|---|---|
| Backend API | `/admin/prompts/*` | ✅ 완성 |
| API 클라이언트 | `lib/api.ts` — `prompts` 객체 | ✅ 완성 (ATL-280) |
| Sidebar 링크 | `components/Sidebar.tsx` | ❌ 미추가 |
| 프롬프트 페이지 | `app/(admin)/prompts/page.tsx` | ❌ 미존재 |

### 관련 백엔드 API

| Method | Endpoint | 설명 |
|---|---|---|
| `GET` | `/admin/prompts` | 프롬프트 목록 조회 (teamId 필터 지원) |
| `POST` | `/admin/prompts` | 새 프롬프트 생성 |
| `GET` | `/admin/prompts/:id` | 특정 프롬프트 조회 |
| `PUT` | `/admin/prompts/:id` | 프롬프트 수정 |
| `DELETE` | `/admin/prompts/:id` | 프롬프트 삭제 |
| `POST` | `/admin/prompts/:id/versions` | 새 버전 생성 |
| `GET` | `/admin/prompts/:id/versions` | 버전 히스토리 조회 |

### 기존 `lib/api.ts` 인터페이스 (참조)

```typescript
export interface Prompt {
  id: string;
  slug: string;
  name: string;
  template: string;
  created_at: string;
  updated_at: string;
}

export interface PromptVersion {
  id: string;
  prompt_id?: string;
  version: string;
  template: string;
  is_active?: boolean;
  created_at: string;
}
```

## 3. Acceptance Criteria

### AC-1: 프롬프트 목록
- **Given** Admin UI에 접속하여 `/prompts`로 이동했을 때
- **When** 페이지가 로드되면
- **Then** 프롬프트 목록(이름, slug, 팀 소속, 최신 버전, 수정일)이 테이블로 표시된다

### AC-2: 프롬프트 생성
- **Given** 목록 페이지에서 "+ New Prompt" 버튼을 클릭했을 때
- **When** slug, name, template을 입력하고 저장하면
- **Then** 목록에 새 항목이 추가되고, 모달이 닫힌다

### AC-3: 프롬프트 에디터 (변수 하이라이팅)
- **Given** 프롬프트 에디터가 열려 있을 때
- **When** template에 `{{variable_name}}` 패턴을 입력하면
- **Then** 해당 패턴이 시각적으로 구분(하이라이팅 또는 badge)된다

### AC-4: 버전 히스토리
- **Given** 특정 프롬프트의 "Versions" 버튼을 클릭했을 때
- **When** 버전 히스토리 패널이 열리면
- **Then** 버전 목록(version, created_at, is_active)이 시간순으로 표시된다

### AC-5: 버전 Diff
- **Given** 버전 히스토리에서 두 버전을 선택했을 때
- **When** diff를 요청하면
- **Then** from/to 두 버전의 template 차이가 시각적으로 표시된다 (라인 단위 before/after)

### AC-6: 버전 롤백
- **Given** 버전 히스토리에서 특정 버전의 "Rollback" 버튼을 클릭했을 때
- **When** 확인하면
- **Then** 해당 버전이 활성화되고, 목록의 "active" 표시가 업데이트된다

### AC-7: Sidebar 링크
- **Given** Admin UI 좌측 Sidebar가 표시된 상태에서
- **When** "Prompts" 링크를 클릭하면
- **Then** `/prompts` 페이지로 이동하고, 링크가 active 스타일로 표시된다

## 5. Technical Approach

### 페이지 구조

기존 `keys/page.tsx`와 `providers/page.tsx` 패턴을 따른다:

```typescript
"use client";
// useQuery for list, useMutation for create/update/delete
// Modal dialogs for create/edit
// Inline panel for version history
```

### 변수 하이라이팅 구현

외부 라이브러리 없이 구현:
- `<textarea>`로 입력받고, 동일 위치에 `<div>`를 겹쳐 HTML로 렌더링
- `{{...}}` 패턴을 `<mark>` 또는 `<span class="bg-amber-200 ...">` 로 변환
- textarea는 투명 배경, 위의 div가 시각적 하이라이팅 제공

### Diff 표시 구현

외부 diff 라이브러리 없이 라인 단위 비교:
- `prompts.diff()` API 응답의 `from.template` / `to.template` 를 라인 단위로 분할
- 간단한 LCS(또는 단순 라인 비교)로 추가/삭제/유지 구분
- 색상: 삭제 `bg-red-100 line-through`, 추가 `bg-green-100`, 유지 기본

## 8. Edge Cases

- 프롬프트 목록이 비어 있을 때 → empty state UI ("No prompts yet" + 생성 버튼)
- 버전이 하나뿐일 때 diff 요청 → diff 버튼 비활성화 또는 "Only one version" 안내
- template이 매우 길 경우 → 에디터 영역 max-height + scroll
- `{{}}` 빈 변수 패턴 → 하이라이팅은 하되 유효성 경고 표시
- 네트워크 오류 시 mutation → toast/알림으로 오류 메시지 표시 (기존 페이지 패턴 준수)
