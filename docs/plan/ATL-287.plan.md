# ATL-287: Admin UI - 차지백/쇼백 리포트 페이지 (/reports)

## 배경 (Background)

LLM Router Gateway의 백엔드에는 팀별 비용 집계를 위한 차지백(Chargeback) / 쇼백(Showback) 리포트 API가 완성되어 있다. 그러나 Admin UI에 해당 페이지가 없어 운영자가 비용 현황을 GUI로 확인할 수 없는 상태다.

- 차지백(Chargeback): 팀별 실제 비용 + 마크업이 포함된 청구 데이터
- 쇼백(Showback): 특정 팀의 모델·프로젝트별 정보성 비용 뷰 (청구 없음)

---

## 현재 상태 (Current State)

### 백엔드 API (완성됨)

| 엔드포인트 | 메서드 | 설명 |
|---|---|---|
| `/admin/reports/chargeback` | GET | 전체 팀 차지백 리포트 (`?period=YYYY-MM[&format=csv]`) |
| `/admin/reports/showback` | GET | 팀별 쇼백 리포트 (`?period=YYYY-MM&team_id=uuid`) |

**Chargeback 응답 구조:**
```typescript
ChargebackReport {
  period: string         // "2026-01"
  generated_at: string
  currency: string
  summary: {
    total_cost_usd: number
    total_tokens: number
    total_requests: number
  }
  by_team: TeamChargebackEntry[]
}

TeamChargebackEntry {
  team_id: string
  team_name: string
  cost_usd: number
  markup_usd: number
  total_charged_usd: number
  tokens: number
  requests: number
  by_model?: ModelBreakdown[]
  by_project?: TagBreakdown[]
}
```

### 프론트엔드 현황

| 항목 | 상태 |
|---|---|
| `lib/api.ts` - `reports` 객체 | 이미 존재하나 타입이 백엔드와 불일치 |
| `components/Sidebar.tsx` - Reports 링크 | 없음 |
| `app/(admin)/reports/page.tsx` | 없음 |

## 범위 (Scope)

### In Scope
- `app/(admin)/reports/page.tsx` 페이지 신규 생성
- `Sidebar.tsx`에 "Reports" 내비게이션 링크 추가
- `lib/api.ts`의 `reports` 관련 타입을 백엔드 실제 응답에 맞게 수정

### Out of Scope
- 리포트 생성 트리거 / 예약(이슈 설명에 언급되었으나 백엔드 미지원)
- PDF 내보내기 (백엔드 CSV만 지원)
- 리포트 목록 조회 (`GET /admin/reports`, `GET /admin/reports/:id` — 백엔드 미구현)
- 마크업 설정 UI (`/admin/billing/markup` — 별도 태스크)

## 인수 조건 (Acceptance Criteria)

### Given-When-Then

**AC-1: 차지백 테이블 조회**
- Given: Admin UI에 로그인한 상태
- When: `/reports` 페이지에 접근하고 기간(YYYY-MM)을 선택
- Then: 팀별 비용(`cost_usd`), 마크업(`markup_usd`), 총 청구액(`total_charged_usd`), 토큰, 요청 수가 테이블로 표시됨

**AC-2: CSV 내보내기**
- Given: `/reports` 페이지에서 차지백 데이터가 로드된 상태
- When: "CSV 내보내기" 버튼 클릭
- Then: `chargeback-{period}.csv` 파일이 브라우저로 다운로드됨

**AC-3: 쇼백 뷰**
- Given: `/reports` 페이지의 "Showback" 탭
- When: 팀을 선택하고 기간을 지정
- Then: 해당 팀의 모델별 비용 breakdown이 표시됨

**AC-4: Sidebar 링크**
- Given: Admin UI 사이드바
- When: 임의의 페이지에서 사이드바 확인
- Then: "Reports" 링크가 표시되고 클릭 시 `/reports`로 이동

## 작업 분해 (Task Breakdown)

### T1. `lib/api.ts` 타입 수정
- 기존 `ChargebackReport`, `ChargebackBreakdown`, `ShowbackReport` 타입 제거
- 백엔드 실제 응답에 맞는 타입 재정의:
  - `ChargebackSummary`, `TeamChargebackEntry`, `ChargebackReport`
  - `ModelBreakdown` (이미 있는 이름과 충돌 → `BillingModelBreakdown` 또는 내부 타입으로 처리)
  - `ShowbackReport`
- `reports.showback()` 시그니처를 `period`를 query param으로 넘기도록 확인

### T2. Sidebar에 "Reports" 링크 추가
- `nav` 배열에 `{ href: "/reports", label: "Reports" }` 추가
- 위치: "Usage" 다음

### T3. `/reports` 페이지 구현
**레이아웃:**
```
/reports
├── Header: "Reports"
├── Period Selector: <input type="month"> → 기본값: 현재 월(YYYY-MM)
├── Tab: [Chargeback] [Showback]
│
├── Chargeback Tab:
│   ├── Summary Cards: 총 비용, 총 토큰, 총 요청
│   ├── CSV 내보내기 버튼 (우측 상단)
│   └── 팀별 테이블:
│       Team | Cost ($) | Markup ($) | Total Charged ($) | Tokens | Requests
│
└── Showback Tab:
    ├── Team Selector (teams API 사용)
    └── (팀 선택 시) 모델별 비용 테이블
```

**기술적 구현:**
- `useQuery` (TanStack Query) 로 데이터 fetch
- CSV 내보내기: `GET /api/admin/reports/chargeback?period=&format=csv` 직접 fetch → Blob download
- 기존 UI 패턴(StatCard, 테이블 스타일) 참조: `usage/page.tsx`, `alerts/page.tsx`

## 엣지 케이스 (Edge Cases)

1. **데이터 없는 기간**: `by_team: []` → "데이터 없음" 상태 표시
2. **Showback 팀 미선택**: 팀 선택 전 안내 메시지 표시
3. **`markup_usd` = 0**: 마크업이 없는 경우 0으로 표시 (숨기지 않음)
4. **빈 `by_model`**: 모델별 breakdown이 없는 팀 → "모델 데이터 없음" 표시
5. **팀 목록 로딩 실패**: 에러 상태 표시
