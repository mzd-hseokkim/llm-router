## 1. 배경 (Background)

LLM Router Gateway의 예산 관리 백엔드(CRUD + 소비 추적)는 완성된 상태이나 관리자 UI가 없어 운영이 불가능하다. 관리자는 현재 API를 직접 호출해야만 팀·키·사용자별 예산 한도를 설정하고 현황을 확인할 수 있다.

이번 작업은 `/budgets` 경로에 Admin Dashboard 페이지를 추가하여 예산 설정과 모니터링을 UI에서 처리할 수 있도록 한다.

---

## 2. 범위 (Scope)

### In Scope
- `admin-ui/app/(admin)/budgets/page.tsx` 신규 생성
- `lib/api.ts`의 `Budget` 인터페이스에 누락 필드 추가 (`id`, `current_spend`)
- Sidebar에 "Budgets" 링크 추가 (`/budgets`)
- 예산 목록 조회 UI (entity_type + entity_id 기반 검색)
- 예산 생성/수정 모달 폼 (POST /admin/budgets upsert)
- 현재 사용량 + Soft/Hard 한도 시각화 (Progress Bar)
- 수동 리셋 버튼 (501 제약 반영: 비활성화 + 안내 툴팁)

### Out of Scope
- 백엔드 신규 엔드포인트 추가 (예: `GET /admin/budgets` 전체 조회)
- 예산 삭제 기능 (백엔드 미구현)
- 개별 업데이트 PUT (upsert 패턴으로 처리)
- 예산 히스토리/변경 이력

## 3. 백엔드 API 현황 분석

| 메서드 | 경로 | 상태 | 비고 |
|---|---|---|---|
| `POST` | `/admin/budgets` | ✅ 완성 | Upsert (생성+수정 통합) |
| `GET` | `/admin/budgets/{entity_type}/{entity_id}` | ✅ 완성 | 특정 엔티티의 예산 목록 |
| `POST` | `/admin/budgets/{id}/reset` | ⚠️ 501 | 미구현 — 기간 종료 시 자동 리셋 |

**주요 제약사항:**
- "전체 예산 목록" (`GET /admin/budgets`) 엔드포인트 없음
- UI는 entity_type + entity_id 를 지정해야 예산 조회 가능
- 리셋 기능은 기간 만료 시 자동 처리되므로 수동 리셋 버튼은 비활성화

**Budget 모델 (Go 백엔드):**
```go
type Budget struct {
    ID           uuid.UUID  // UUID
    EntityType   string     // "key" | "user" | "team" | "org"
    EntityID     uuid.UUID  // 엔티티 ID (Virtual Key ID 등)
    Period       string     // "hourly" | "daily" | "weekly" | "monthly" | "lifetime"
    SoftLimitUSD *float64   // 소프트 한도 (경고만, nil이면 미설정)
    HardLimitUSD *float64   // 하드 한도 (차단, nil이면 미설정)
    CurrentSpend float64    // 현재 사용량 (USD)
    PeriodStart  time.Time
    PeriodEnd    time.Time
}
```

## 5. 수용 기준 (Acceptance Criteria)

### AC-1: 예산 목록 조회
- Given: entity_type(dropdown)과 entity_id(UUID 입력)를 지정하고 조회
- When: "조회" 버튼 클릭 또는 자동 조회
- Then: 해당 엔티티의 예산 목록이 테이블로 표시된다 (기간, 사용량, 한도, 바 차트)

### AC-2: 사용량 바 차트
- Given: 예산 목록이 표시된 상태
- Then: 각 행에 "current_spend / hard_limit_usd (또는 soft_limit_usd)" 비율의 Progress Bar 표시
- Then: soft 한도 초과 시 주황색, hard 한도 초과 시 빨간색 표시

### AC-3: 예산 생성/수정
- Given: "새 예산" 버튼 클릭
- Then: entity_type, entity_id, period, soft_limit, hard_limit 입력 모달이 열린다
- When: 저장 시 `POST /admin/budgets`(upsert) 호출
- Then: 목록이 갱신된다

### AC-4: 수동 리셋 (비활성화)
- Given: 예산 행의 "리셋" 버튼
- Then: 버튼이 비활성화 상태로 표시되고, hover 시 "기간 종료 시 자동으로 리셋됩니다" 툴팁 표시

### AC-5: Sidebar 링크
- Given: Admin Dashboard 어느 페이지에서나
- Then: Sidebar에 "Budgets" 메뉴가 표시되고 `/budgets`로 이동한다

## 7. UI 설계

### 페이지 레이아웃

```
┌─────────────────────────────────────────────────┐
│  예산 관리                      [+ 새 예산]       │
│  팀·키·사용자별 예산 한도 설정                     │
├─────────────────────────────────────────────────┤
│  Entity Type [key ▼]  Entity ID [________] [조회]│
├─────────────────────────────────────────────────┤
│  기간     │ Soft 한도 │ Hard 한도 │ 사용량  │ 리셋 │
├───────────┼──────────┼──────────┼─────────┼──── │
│  monthly  │  $50.00  │ $100.00  │         │  🔒 │
│  ████████░ $82.50 / $100.00 (82%)         │     │
└─────────────────────────────────────────────────┘
```

### 모달: 예산 생성/수정

```
┌────────────────────────────────┐
│  예산 설정                      │
├────────────────────────────────┤
│  Entity Type  [key        ▼]   │
│  Entity ID    [UUID_______]    │
│  기간         [monthly    ▼]   │
│  Soft 한도 ($) [50.00______]   │
│  Hard 한도 ($) [100.00_____]   │
│                                │
│         [취소]  [저장]          │
└────────────────────────────────┘
```

### Progress Bar 색상 규칙

- 0~60%: `bg-green-500`
- 60~80%: `bg-yellow-400`
- 80~100%: `bg-orange-500` (Soft 초과 시)
- 100%+: `bg-red-500` (Hard 초과 또는 초과 임박)

## 8. 위험 요소 (Risks)

| 위험 | 영향 | 대응 |
|---|---|---|
| 전체 예산 조회 엔드포인트 없음 | 중 | 검색 폼으로 entity 지정 후 조회하는 UX로 대응 |
| `reset` 501 상태 | 低 | 버튼 비활성화 + 안내 툴팁 |
| entity_id가 UUID 형식 | 低 | 입력 필드에 UUID 유효성 검사 추가 |
| `soft_limit_usd`, `hard_limit_usd`가 null 가능 | 低 | null 체크 후 "미설정" 표시 |

---

## 9. 경계 조건 (Edge Cases)

- entity_id에 해당하는 예산이 없는 경우: 빈 상태(empty state) + "예산 없음" 안내
- soft_limit만 설정되고 hard_limit이 없는 경우: 한도 없음으로 표시, progress bar는 soft_limit 기준
- current_spend가 hard_limit 초과인 경우: 100% 초과 표시 (ex: 110%)
- entity_id가 유효하지 않은 UUID 입력: 폼 유효성 검사 오류 표시
- API 오류 시: 에러 메시지 표시, 빈 상태 유지
