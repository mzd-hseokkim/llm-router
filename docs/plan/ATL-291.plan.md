## Background

LLM Router Gateway는 OpenAI, Anthropic, Gemini 등 여러 LLM Provider에 대한 Circuit Breaker를 관리한다.
Provider 오류 발생 시 서킷이 OPEN 상태가 되어 해당 Provider로의 요청이 차단되고, 이후 HALF_OPEN → CLOSED 순서로 복구된다.

현재 Dashboard에서 Circuit Breaker 상태를 표시하지만, 수동 리셋 기능이 없다.
Keys 페이지에 Circuit Breaker 섹션을 추가하여 운영자가 Provider별 서킷 상태를 확인하고 수동으로 리셋할 수 있어야 한다.

## Scope

### In Scope

- Keys 페이지(`app/(admin)/keys/page.tsx`)에 Circuit Breaker 패널 추가

- Dashboard 수정 (기존 읽기 전용 표시 유지)
- Circuit Breaker 임계값 설정 UI

## Acceptance Criteria

### AC-1: Circuit Breaker 목록 표시
- **Given** Keys 페이지를 열면
- **When** Gateway에서 Circuit Breaker 데이터를 반환할 때
- **Then** Provider별 이름, 상태(CLOSED/OPEN/HALF_OPEN), 실패 횟수가 뱃지로 표시된다

### AC-2: 상태 색상 구분
- **Given** Circuit Breaker 상태가 있을 때
- **Then** CLOSED는 초록색, OPEN은 빨간색, HALF_OPEN은 노란색으로 표시된다

### AC-3: 수동 리셋
- **Given** Circuit Breaker가 OPEN 또는 HALF_OPEN 상태일 때
- **When** 운영자가 Reset 버튼을 클릭하면
- **Then** `POST /admin/circuit-breakers/{provider}/reset` API가 호출되고, 상태가 CLOSED로 갱신된다

### AC-4: 리셋 버튼 비활성화
- **Given** Circuit Breaker가 CLOSED 상태일 때
- **Then** Reset 버튼이 비활성화(disabled)된다

### AC-5: 데이터 없음 처리
- **Given** 추적 중인 Circuit Breaker가 없을 때
- **Then** "No circuit breakers tracked" 메시지가 표시된다

### AC-6: 자동 갱신
- **Then** 30초마다 Circuit Breaker 상태가 자동으로 갱신된다

## Risks & Considerations

| 위험 | 영향 | 대응 |
|------|------|------|
| Circuit Breaker 데이터가 없는 경우 | 낮음 | 빈 상태 UI 처리 |
| Reset API 실패 (네트워크 오류 등) | 낮음 | 에러 메시지 표시 후 계속 진행 |
| 대시보드와 UI 중복 | 낮음 | 대시보드는 읽기 전용 요약, Keys 페이지는 리셋 가능한 상세 뷰로 역할 구분 |

---

## Edge Cases

- Circuit Breaker 상태가 `null` 또는 알 수 없는 값인 경우 → 기본 스타일 적용
- reset_time이 과거인 경우 → 그대로 표시 (자동 전환은 서버 로직)
- 동시에 여러 Provider를 Reset하는 경우 → 각 버튼 독립적으로 동작

## 참고 코드

### 이미 구현된 API 타입 (lib/api.ts:858-864)
```typescript
export interface CircuitBreaker {
  provider: string;
  state: string;        // "closed" | "open" | "half_open"
  failure_count: number;
  last_failure?: string;
  reset_time?: string;
}

export const circuitBreakers = {
  list: () => apiFetch<{ circuit_breakers: CircuitBreaker[] }>(
    "/circuit-breakers").then((r) => r.circuit_breakers ?? []),
  reset: (provider: string) => apiFetch<{ provider: string; state: string; message: string }>(
    `/circuit-breakers/${provider}/reset`, { method: "POST" }),
};
```
